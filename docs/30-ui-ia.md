> 화면 구조 및 네비게이션 설계

---

## 1. 화면 구조

### 1.1 전체 사이트맵

```
/                           # 랜딩/로그인
├── /lobby                  # 로비 (방 목록)
│   └── /lobby/create       # 방 생성 모달
├── /table/:id              # 테이블 (게임)
│   ├── /table/:id/spectate # 관전 모드
│   └── /table/:id/history  # 핸드 히스토리
├── /profile                # 프로필
│   ├── /profile/settings   # 설정
│   └── /profile/history    # 내 게임 기록
└── /auth                   # 인증
    ├── /auth/login         # 로그인
    ├── /auth/register      # 회원가입
    └── /auth/forgot        # 비밀번호 찾기
```

### 1.2 화면별 접근 권한

| 화면 | 비로그인 | 로그인 | 착석 |
|------|---------|--------|------|
| 랜딩 | ✓ | ✓ | - |
| 로비 | - | ✓ | - |
| 테이블 (관전) | - | ✓ | - |
| 테이블 (플레이) | - | ✓ | ✓ |
| 프로필 | - | ✓ | - |

## 3. 네비게이션 흐름

### 3.1 메인 플로우

```
[Login] → [Lobby] → [Join Room] → [Table] → [Seat] → [Play]
                         ↓
                    [Spectate]
```

### 3.2 게임 종료 플로우

```
[Table] → [Leave] → [Lobby]
              ↓
         [Continue] → [Next Hand]
```

### 3.3 재접속 플로우

```
[Disconnect] → [Reconnecting...] → [Table Snapshot] → [Resume]
                     ↓
              [Failed] → [Lobby]
```

## 4. 상태별 화면 처리

### 4.1 연결 상태

| 상태 | UI 표시 | 사용자 액션 |
|------|---------|-----------|
| Connected | 없음 | 정상 플레이 |
| Reconnecting | 상단 배너 "재연결 중..." | 액션 비활성화 |
| Disconnected | 모달 "연결 끊김" | 재연결 버튼 |

### 4.2 로딩 상태

| 화면 | 로딩 UI |
|------|---------|
| 로비 | 스켈레톤 카드 |
| 테이블 | 스피너 + "테이블 로딩 중..." |
| 액션 | 버튼 스피너 |

### 4.3 에러 상태

| 에러 | UI 표시 |
|------|---------|
| 네트워크 에러 | 토스트 "네트워크 오류" |
| 권한 에러 | 모달 "접근 권한 없음" |
| 서버 에러 | 토스트 "서버 오류, 잠시 후 재시도" |

## 5. 반응형 브레이크포인트

| 브레이크포인트 | 너비 | 레이아웃 |
|--------------|------|---------|
| Mobile | < 640px | 단일 컬럼, 축소 테이블 |
| Tablet | 640-1024px | 2컬럼 로비, 표준 테이블 |
| Desktop | > 1024px | 3컬럼 로비, 확장 테이블 |

## 6. 접근성 (A11y)

### 6.1 키보드 네비게이션

| 키 | 동작 |
|----|------|
| Tab | 포커스 이동 |
| Enter | 선택/확인 |
| Escape | 모달 닫기 |
| Arrow Keys | 슬라이더/선택 |

### 6.2 스크린 리더

- 모든 버튼에 aria-label
- 상태 변경 시 aria-live 알림
- 카드 이미지에 alt 텍스트
